# 曼谷旅伴行程小工具規格

## 專案目標

製作一個手機優先的曼谷旅遊網頁小工具，讓四位旅伴可以快速知道：

- 今天要去哪裡。
- 每個地點大概會花多少時間。
- 每個地點大概會花多少錢。
- 今天自己負責什麼角色。
- 今天有沒有被隨機抽中成為「煙捲」。
- 候補行程是否要加入或移出今日行程。
- 自己目前花了多少錢、還剩多少預算。

這趟旅程每個人的預算是 **NT$20,000**，已經扣除機票和酒店。

## 旅程設定

- 旅程日期：2026/7/5 到 2026/7/10。
- 2026/7/5 是出發搭飛機、抵達與移動日。
- 2026/7/10 是回程搭飛機與移動日。
- 2026/7/6 到 2026/7/9 是主要遊玩日。
- 總共有四位旅伴。
- 四位旅伴名稱：液蠔、🏠🍋、疣法、🐎🐑。
- 每位旅伴各自有 NT$20,000 預算。

## 角色設定

四個角色每天輪替：

- 決策者：負責當天最終決定，例如去哪裡、吃什麼、要不要加入候補景點。
- 背包人：負責公共物品、票券、藥品、備用品。
- 相機人：負責拍照、記錄、整理旅行影像。
- 叫車的：負責 Grab、Bolt、計程車溝通、目的地與路線確認。

每位旅伴打開連結後會先抽出自己的身份，之後每天會看到自己當天的角色。

另外每天會額外隨機抽出一位旅伴成為「煙捲」。

- 煙捲：氣氛補給官，負責提醒大家休息、補水、帶零食、找放鬆點、放歌單，以及記錄當天好笑片段。

「煙捲」不是四個固定輪替角色之一，而是每天額外抽出的隱藏角色。同一個人當天仍然會保有原本的固定角色。

## 第一版功能範圍

第一版會做：

- 手機優先的單頁網頁。
- 打開連結後先抽旅伴身份。
- 今日頁：顯示今天日期、今天角色、今日行程、候補地點。
- 行程頁：顯示 2026/7/5 到 2026/7/10 每一天安排。
- 候補池：放大家正在收集的景點、餐廳、活動。
- 我的預算：記錄自己的支出、看已花費與剩餘金額。
- 角色頁：只顯示當日四位旅伴的角色分配，以及當天隨機抽中的「煙捲」。
- 候補行程可加入今日行程，也可從今日行程移除。
- 使用手機瀏覽器本機儲存，不需要登入。

第一版暫時不做：

- 會員登入。
- 多人即時同步。
- 每個旅伴自己新增景點。
- 後端資料庫。
- 分帳功能。
- 自動排路線。
- 即時天氣、匯率或地圖 API。

## 頁面設計

### 1. 抽身份頁

第一次打開連結時，每個人會先抽出自己的旅伴身份。

身份使用：

- 液蠔
- 🏠🍋
- 疣法
- 🐎🐑

四個身份會使用不同顏色，讓角色分配更容易辨識。

### 2. 今日頁

這會是大家最常看的頁面。

今日頁會顯示：

- 今天是旅程第幾天。
- 今天是移動日還是主要遊玩日。
- 自己今天的角色。
- 自己今天有沒有被抽中成為「煙捲」。
- 今天預排的行程。
- 每個地點預估停留時間。
- 每個地點預估花費。
- 候補景點、餐廳或活動。
- 今日預估總花費。

### 3. 行程頁

行程頁會顯示六天日期：

- 7/5 出發與抵達日。
- 7/6 主要遊玩日。
- 7/7 主要遊玩日。
- 7/8 主要遊玩日。
- 7/9 主要遊玩日。
- 7/10 回程日。

每個行程項目會包含：

- 名稱。
- 類型：景點、餐廳、咖啡廳、購物、活動、交通或其他。
- 區域。
- 預估停留時間。
- 預估每人花費。
- 備註。
- 狀態：預排、候補或尚未排入。

### 4. 候補池

候補池用來放大家目前正在收集的景點、餐廳與活動。

你會先統一整理大家想去的地方，再放進網頁資料。

候補池可以依照以下條件查看：

- 類型。
- 區域。
- 優先度。
- 預估花費。
- 預估停留時間。

當天的決策者可以依照體力、天氣、時間、預算，決定要不要加入候補地點。

候補池中的地點可以點「加入今日」放入今日行程；已在今日行程中的地點可以點「從今日移除」，移除後會回到可加入狀態。

### 5. 我的預算

每位旅伴都有自己的預算頁。

預算頁會顯示：

- 起始預算：NT$20,000。
- 已花費金額。
- 剩餘金額。
- 預算使用比例。
- 支出紀錄。

每筆支出可以填：

- 日期。
- 類別：食物、交通、門票、購物、活動、咖啡、便利商店或其他。
- 金額。
- 地點，可選填。
- 備註，可選填。

這些支出只存在自己的手機瀏覽器裡，不會同步到其他人的手機。

### 6. 角色頁

角色頁只顯示當日的角色輪替，不顯示整趟旅程所有日期。

第一版建議讓角色每天都輪替，包含出發日與回程日，這樣表格最簡單，也比較公平。

角色頁也會顯示當日被隨機抽中的「煙捲」。這個隨機結果會依照日期固定產生，讓所有人看到同一天時結果一致，不會每次重新整理就改變。

## 資料儲存方式

第一版使用手機瀏覽器本機儲存。

會存在手機裡的資料：

- 自己抽到的旅伴身份。
- 自己輸入的支出紀錄。
- 自己調整過的今日行程。
- 一些介面偏好，例如上次看的頁面。

不會存在手機裡、而是寫在網頁資料裡的內容：

- 旅程日期。
- 角色設定。
- 煙捲每日隨機規則。
- 景點、餐廳、活動資料。
- 每日行程。

## 後續需要補的內容

這些資料之後可以慢慢補，不影響第一版架構：

- 四位旅伴的名字或暱稱。
- 想去的景點。
- 想吃的餐廳。
- 想安排的活動。
- 每個地點的預估花費。
- 每個地點的預估停留時間。
- 飛機抵達與回程時間。

## 驗收標準

第一版完成後，需要確認：

- 手機上看起來清楚好用。
- 第一次打開可以抽旅伴身份。
- 四位旅伴名稱會顯示為液蠔、🏠🍋、疣法、🐎🐑，且顏色互相區分。
- 每天四個人都會拿到不同角色。
- 每天會額外抽出一位「煙捲」，且同一天重新整理後結果不會改變。
- 角色頁只顯示當日角色分配。
- 候補行程可以加入今日行程，也可以從今日行程移除。
- 7/5 和 7/10 會被標示為搭飛機與移動日。
- 每個人的預算都是 NT$20,000。
- 可以新增、修改、刪除自己的支出。
- 重新整理網頁後，自己的支出紀錄還在。
- 已花費與剩餘預算會正確計算。
- 候補池就算還沒填滿，也看起來不是壞掉的頁面。
