# 曼谷旅伴行程小工具 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立一個手機優先的靜態網頁，讓四位旅伴抽身份、查看每日角色與煙捲、瀏覽曼谷行程/候補池，並在自己的手機記錄 NT$20,000 個人預算支出。

**Architecture:** 使用無後端的單頁靜態網頁。`app.js` 保存旅程資料、角色演算法、localStorage 狀態、畫面渲染與互動；`styles.css` 負責手機優先視覺與響應式版面；`index.html` 提供語意結構與底部導覽。

**Tech Stack:** HTML, CSS, vanilla JavaScript, browser localStorage, Python static server for local verification.

---

## File Structure

- Create `index.html`: app shell, setup screen, main screen regions, bottom tabs.
- Create `styles.css`: mobile-first layout, cards, tabs, forms, budget indicators, desktop fallback.
- Create `app.js`: trip data, role rotation, deterministic daily smoke-roll role, localStorage persistence, render functions, expense CRUD.
- Modify `README.md`: add how to open and test the static app.
